## 1. Ядро: фреймворк и язык

### Next.js 15 (App Router)
- **Что:** React-фреймворк. Даёт маршрутизацию, рендеринг на сервере (SSR), серверные функции, сборку.
- **Зачем здесь:** хороший SEO для магазина, серверные эндпоинты, лёгкий деплой на Vercel.
- **Где:** вся папка `src/app/` (страницы, layout, API). Конфиг — `next.config.mjs`.

### React 19
- **Что:** библиотека для интерфейсов из компонентов.
- **Зачем:** весь UI — это React-компоненты.
- **Где:** `src/components/*`. Хуки: `useState` (состояние), `useEffect` (побочные эффекты), `useMemo` (кэш вычислений), `useRef` (ссылка/таймер), `useSyncExternalStore` (подписка на внешнее хранилище), `useContext`.

### TypeScript
- **Что:** JavaScript с типами (ловит ошибки до запуска).
- **Зачем:** надёжность, автодополнение.
- **Где:** все `.ts/.tsx`. Типы данных — `src/lib/types.ts`. Конфиг — `tsconfig.json`.

## 2. Ключевые концепции Next.js (важно понять)

### Server Components vs Client Components
- По умолчанию компоненты — **серверные** (рендерятся на сервере, быстрее, видят секреты).
- Если компонент использует интерактив (`useState`, события, `localStorage`) — вверху файла пишем **`"use client"`** → он клиентский.
- **Где:** почти все наши компоненты — клиентские (`"use client"`), т.к. интерактив (корзина, фильтры).

### App Router (файловая маршрутизация)
- Папка = маршрут. `app/page.tsx` → `/`, `app/admin/page.tsx` → `/admin`.
- `layout.tsx` — обёртка вокруг страниц (общая для группы).
- **Где:** `src/app/` (главная, `admin/*`, `api/*`).

### Route Handlers (API)
- **Что:** серверные эндпоинты (бэкенд внутри Next).
- **Где:** `src/app/api/order/route.ts`, `src/app/api/review/route.ts` (принимают POST, шлют в Telegram).

### Переменные окружения
- `NEXT_PUBLIC_*` — видны в браузере (публичные).
- Без префикса — **только сервер** (секреты: `SUPABASE_SERVICE_ROLE_KEY`, `TELEGRAM_BOT_TOKEN`).
- **Где:** `.env.local` (локально, в git не попадает), Vercel → Settings → Environment Variables (прод).

## 3. Стилизация

### CSS-переменные (custom properties) + глобальные классы
- **Что:** дизайн-токены (цвета, шрифты) и адаптивные значения в CSS-переменных.
- **Зачем:** один источник правды для темы + адаптив без переписывания компонентов (меняем переменную в медиазапросе).
- **Где:** `src/app/globals.css` (`:root { --accent, --py, --menu-cols ... }`), инлайн-стили компонентов ссылаются на них.

### next/font
- **Что:** подключение шрифтов Google (self-hosted, без внешних запросов, без «прыжков» текста).
- **Где:** `src/app/layout.tsx` (Cormorant Garamond + Jost).

## 4. Состояние на клиенте (без БД)

### React Context
- **Что:** способ делиться состоянием между компонентами без передачи через пропсы.
- **Где:** `CartContext` (корзина), `AdminAuthContext` (демо-вход/роль в админке).

### useSyncExternalStore + localStorage
- **Что:** паттерн «внешнее хранилище» — данные в `localStorage`, компоненты подписаны и обновляются.
- **Зачем:** временные демо-данные админки и категорий, пока не подключили БД полностью.
- **Где:** `src/features/admin/createLocalStore.ts` (фабрика), `categoriesStore`, `stores.ts` (товары/ингредиенты/акции/промокоды), `priceHistory.ts`.

### localStorage
- **Что:** хранилище в браузере (переживает перезагрузку).
- **Где:** корзина (`ss_cart_v1`), демо-данные админки.

## 5. Бэкенд и сервисы

### Supabase
- **Что:** «бэкенд из коробки» — база Postgres + авторизация + хранилище файлов, с бесплатным тарифом.
- **Зачем:** одна бесплатная платформа под БД/Auth/Storage.
- **Где:** клиенты `src/lib/supabase/{client,server,admin}.ts`; схема `supabase/migrations/`; пакеты `@supabase/supabase-js`, `@supabase/ssr`.

### PostgreSQL (внутри Supabase)
- **Что:** реляционная БД (таблицы, связи).
- **Ключевые понятия у нас:**
  - **Таблицы и связи** (`products` ↔ `ingredients` через `product_ingredients`).
  - **RLS (Row Level Security)** — правила доступа к строкам прямо в БД (читать публично, писать — staff, удалять — только admin).
  - **Функции и триггеры** (`handle_new_user` — при входе через Google сверяет email с белым списком).
- **Где:** `supabase/migrations/20260610120000_init.sql`.

### service_role vs anon ключ
- **anon** (`NEXT_PUBLIC_SUPABASE_ANON_KEY`) — для браузера, ограничен RLS.
- **service_role** (`SUPABASE_SERVICE_ROLE_KEY`) — полный доступ, **обходит RLS**, только на сервере.
- **Где:** `src/lib/supabase/admin.ts` (service_role, серверный).

### Telegram Bot API
- **Что:** отправка сообщений ботом в группу через HTTP-запрос.
- **Где:** `src/lib/telegram.ts` + API-роуты заказов/отзывов.

## 6. Инструменты и инфраструктура

- **Node.js / npm** — среда выполнения и менеджер пакетов (`package.json`).
- **Git / GitHub** — версионирование, репозиторий (SSH-доступ).
- **Vercel** — хостинг: авто-деплой при `git push`, серверные функции, переменные окружения.
- **ESLint** (`next lint`) и **tsc** (`tsc --noEmit`) — проверка кода и типов.
- **pg** (одноразово) — применение SQL-миграций скриптом. **sharp** (планируется) — оптимизация картинок.

## 7. Планируется (в бэклоге, см. STATUS.md)
- **Cloudflare R2** — S3-совместимое хранилище для картинок (дешевле/просторнее Storage) + **sharp** для конвертации в **WebP**.
- **OpenStreetMap / Photon** — автоподсказки адреса и расчёт расстояния для доставки.
- **Google OAuth** (через Supabase Auth) — вход в админку.
